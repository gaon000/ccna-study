ip 주소는 32자리 2진수로 이루어져있음(octet)


네트워크는 하나의 브로드캐스트 영역

ip주소는 네트워크 부분과 호스트 부분으로 나뉜다.

같은 네트워크에서는 네트워크 부분은 모두 같고 호스트 부분은 모두 달라야 된다.

네트워크 부분과 호스트 부분을 나누는 방법은 ip주소의 클래스에 따라서 다르다(네트워크 크기에 따른 구분).

class:
	class A: 호스트 수가 가장 많음, 32개의 2진수중 항상 맨 앞 하나는 0일때임, 맨 앞의 8비트가 네트워크 부분, 나머지 뒷부분이 호스트 부분, 호스트 부분이 0.0.0인 경우는 호스트 주소가 아닌 네트워크 전체를 나타내기 때문에 사용x, 255.255.255는 브로드캐스트 주소이기 때문에 호스트 주소로 불가능, 범위: 1~126(127은 예약됨), 16777214개
	
	class B: 맨 앞이 반드시 10으로 시작됨, 앞의 16비트가 네트워크 부분, 나머지가 호스트부분을 나타냄, 범위: 128~192, 65534개
	
	class C: 맨 앞이 110으로 시작됨, 앞의 24비트가 네트워크부분, 나머지가 호스트부분을 나타낸다, 범위: 192~223, 254개
	
	class D: 멀티캐스트용 주소, 범위: 224~239
	
	class E: 예약됨, 범위: 240~255
